Client Side Factors That Affect Website Performance

The device being used by the user can affect website performance. This is because the device may only support a limited number of browsers and the browsers may not be particularly good ones. If the user is using a mobile device they may have issues with some sites because not all websites have mobiles versions which would result in the website and it's elements not being scaled properly and the experience may be slow because the mobile device has to load a full desktop website.

The hardware specifications of the user's device is a major factor which can affect the performance of a website. This is because if the components such as the CPU and GPU are slow then this will cause the website to have longer loading times and once the website has been loaded if there are a large amount of css animations or javascript scripts there could be slow downs.

The user's choice of browser may affect the performance of websites. This is because the may not support specific features which the website makes use of and may cause the website to not perform properly. Some browsers may not support hardware acceleration which could cause slowdowns on websites which are graphically demanding.

Server Side Factors That Affect Website Performance

The web server's capacity can be a large factor in a websites performance. This is because a web server can only handle a certain number of requests at once and if the server gets overwhelmed by requests then it can potentially crash or slow down. This would result in slowdowns for the user when trying to load the webpage and trying to do anyone on the website which requires communication with the web server.

The bandwidth available to the server can also affect the performance of the website. This is because if the web server does not have sufficient bandwidth then it will take longer for the web server to send the website files to the client computer causing longer load times for the user.
